# 长期记忆优化配置方案

## 配置完成时间
2026-03-28

## 核心目标
解决项目执行的**长期记忆问题**，包括：
- 跨会话记忆（claude-mem）
- 项目执行记忆（planning-with-files）

## 优化策略

### 原则
保留两者核心价值，降低噪声干扰

### 具体优化

**1. UserPromptSubmit - 精简提醒**
- 原：显示完整计划文本
- 现：只显示进度统计（3/5 完成）
- 效果：一行提醒，零噪声

**2. PreToolUse - 降低频率**
- 原：每次 Write|Edit|Bash 都显示 30 行
- 现：只在 Write|Edit 时显示当前阶段（5行）
- 效果：减少 50% 触发，内容减少 80%

**3. PostToolUse - 移除重复**
- 原：planning 提醒 + mem 观察捕获
- 现：只保留 mem 观察捕获
- 效果：消除双重提醒

**4. Stop - 保持不变**
- planning 完成检查 + mem 摘要生成
- 两者顺序执行，互不干扰

```
    │                   │
┌───▼────────┐   ┌──────▼──────┐

    │                   │

```

## 互补效果

| 场景 | claude-mem | planning-with-files |
|------|-----------|-------------------|
| 新项目启动 | 注入相似项目经验 | 创建任务结构 |
| 执行过程 | 自动记录观察 | 追踪当前阶段 |
| 遇到错误 | 搜索历史解决方案 | 记录错误+尝试 |
| 阶段完成 | 捕获关键决策 | 更新状态+原因 |
| 会话结束 | AI 压缩摘要 | 检查完成状态 |
| 下次启动 | 恢复项目上下文 | 继续未完成任务 |
| 3个月后 | 语义搜索"为什么" | 查看决策历史 |

## 使用方式

### 日常工作流
1. 启动会话 → claude-mem 自动注入历史
2. 复杂任务 → `/plan-zh` 创建规划
3. 执行过程 → 自动提醒当前阶段
4. 会话结束 → 自动检查+摘要

### 长期项目
- task_plan.md：记录"为什么这样做"
- findings.md：存档研究发现
- progress.md：追踪执行历史
- claude-mem：自动压缩成可搜索记忆
